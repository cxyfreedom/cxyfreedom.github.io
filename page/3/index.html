Python 中 Redis 连接池原理

2019-04-25 18:33
Python
约 764 字 预计阅读 2 分钟

通常情况下，我们进行 redis 操作时，会创建一个连接，并基于这个连接进行操作。操作完成后，释放该连接。但是当并发较高时，频繁的创建和释放会对性能造成影响。连接池的原理是通过预先创建多个连接。进行获取已经创建好的连接进行操作，操作完成之后，不会进行释放，可以用于后续其他redis的操作。
